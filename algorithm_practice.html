Q1. Big O / Omega / Theta 판별 ★
기출 빈출 유형 — True / False로 답하고 이유를 설명하시오
1
다음 중 옳은 것은? (True / False)
(a) n³ + 100n ∈ O(n³)    (b) n² ∈ O(n³)    (c) 2n ∈ O(n²)    (d) n³ ∈ O(n²)
Big O ›
문제

각 항목에 대해 True/False를 판단하고 이유를 설명하시오.

풀이

Big O 정의: ∃c > 0, ∃N ≥ 0 s.t. g(n) ≤ c × f(n), ∀n ≥ N

항목답이유
(a) n³+100n ∈ O(n³)Truen ≥ 1에서 100n ≤ 100n³ 이므로 n³+100n ≤ 101n³. c=101, N=1
(b) n² ∈ O(n³)Truen ≥ 1에서 n² ≤ n³. c=1, N=1
(c) 2n ∈ O(n²)Truen ≥ 2에서 2n ≤ n². c=1, N=2
(d) n³ ∈ O(n²)Falsen³ ≤ cn² → n ≤ c 인데, n은 한없이 커지므로 모순
핵심: Big O는 상한! g(n)이 f(n)보다 차수가 같거나 낮으면 항상 True
+
다음 중 옳은 것은? (True / False)
(a) n² + 10n ∈ Ω(n²)    (b) n ∈ Ω(n²)    (c) n² + 10n ∈ Θ(n²)    (d) n ∈ Θ(n²)
Ω / Θ ›
문제

각 항목에 대해 True/False를 판단하고 이유를 설명하시오.

풀이
항목답이유
(a) n²+10n ∈ Ω(n²)Truen ≥ 0에서 n²+10n ≥ n². c=1, N=0
(b) n ∈ Ω(n²)Falsen ≥ cn² → 1/c ≥ n, n은 한없이 커지므로 모순
(c) n²+10n ∈ Θ(n²)TrueO(n²)이면서 Ω(n²) → Θ(n²) 성립
(d) n ∈ Θ(n²)FalseΩ(n²)가 불성립하므로 Θ(n²)도 불성립
Θ = O ∩ Ω. 둘 다 성립해야 Θ!
Q2. small o 판별 ★
Big O와 small o의 차이가 핵심!
2
다음 중 옳은 것은? (True / False)
(a) n ∈ o(n²)    (b) n² ∈ o(n²)    (c) n ∈ o(5n)    (d) n³ ∈ o(n²)
small o ›
문제

small o 정의: ∀c > 0, ∃N ≥ 0 s.t. g(n) ≤ c × f(n), ∀n ≥ N
(모든 c에 대해 성립해야 함! Big O는 어떤 c 하나만 찾으면 됨)

풀이
항목답이유
(a) n ∈ o(n²)True임의의 c > 0에 대해, N = ⌈1/c⌉ 잡으면 n ≥ N에서 n ≤ cn² 성립
(b) n² ∈ o(n²)Falsec = 0.5로 잡으면 n² ≤ 0.5n² 는 불가능
(c) n ∈ o(5n)Falsec = 0.1로 잡으면 n ≤ 0.5n 은 불가능 (같은 성장률)
(d) n³ ∈ o(n²)Falsen³ ≤ cn² → n ≤ c, n은 한없이 커지므로 모순
small o vs Big O 핵심 차이: Big O는 ∃c (하나 찾으면 됨), small o는 ∀c (모든 c에 대해). 같은 차수면 Big O는 가능, small o는 불가!
Q3. Master Theorem 적용 ★
점화식 보고 케이스 판별 → 시간복잡도 결론
3
다음 점화식의 시간복잡도를 Master Theorem으로 구하시오.
(a) T(n) = 4T(n/2) + n    (b) T(n) = 2T(n/2) + n lg n    (c) T(n) = 3T(n/4) + n
Master Thm ›
문제

T(n) = aT(n/b) + f(n) 형태. log_b(a) 계산 후 f(n)과 비교해서 Case 결정.

풀이

(a) T(n) = 4T(n/2) + n

a=4, b=2, log₂4 = 2

f(n) = n = O(n^(2-1)) = O(n^(log_b a - 1)) → Case 1

T(n) = Θ(n²)

(b) T(n) = 2T(n/2) + n lg n

a=2, b=2, log₂2 = 1

f(n) = n lg n. n¹ vs n lg n → n lg n이 n보다 크고 n^(1+ε)보다는 작음

→ Master Theorem 적용 불가 (어느 케이스에도 해당 안 됨)

T(n) = Θ(n lg² n) — Master Theorem 예외 케이스! 직접 전개해야 함

(c) T(n) = 3T(n/4) + n

a=3, b=4, log₄3 ≈ 0.79

f(n) = n = Ω(n^(0.79+0.21)) → Case 3

T(n) = Θ(n)

코드 빈칸 ›

mergeSort(int n, keytype[] S) { if (n > 1) { const int h = floor(n/2), m = n - h; copy S[1:h] to U[1:h]; copy ① to V[1:m]; mergeSort(h, U); mergeSort(②); merge(h, m, U, V, S); } } merge(int h, int m, keytype[] U, V, S) { index i=1, j=1, k=1; while (i <= h && j <= m) { if (U[i] < V[j]) { S[k] = U[i]; i++; } else { S[k] = V[j]; j++; } k++; } if (i > h) copy V[j:m] to S[k:h+m]; else copy ③ to S[k:h+m]; } partition(index low, index high) { pivotItem = S[low]; j = low; for (i = low+1; i <= high; i++) if (S[i] < ④) exchange S[i] and S[++j]; exchange S[low] and S[j]; return j; }
정답

②m, V오른쪽 부분(크기 m, 배열 V)을 재귀 정렬
③U[i:h]V가 소진됐으므로 U의 나머지를 그대로 복사

Q10. 종합 단답형
시험에 단답으로 나올 만한 것들 모음
A
MergeSort와 QuickSort에서 Combine 단계의 차이는?
개념 ›
정답

MergeSort: Combine 단계 필요 — 두 정렬된 배열을 merge() 함수로 합침

QuickSort: Combine 단계 불필요 — partition 후 이미 pivot 기준으로 분리됨. 재귀만 하면 끝!

QuickSort는 "No combine step needed" 가 핵심 특징!
B
Optimal BST 테이블을 채울 때 대각선 방향으로 채우는 이유는?
개념 ›
정답

A[i][j]를 계산하려면 A[i][k-1](왼쪽 서브트리)과 A[k+1][j](오른쪽 서브트리)가 먼저 계산되어 있어야 함.

이 두 값은 항상 A[i][j]보다 구간 크기가 작음 (j-i+1 보다 작음).

따라서 구간 크기 1 → 2 → ... → n 순으로, 즉 대각선 방향으로 채워야 참조할 때 값이 이미 있음!

C
D&C를 사용하면 안 되는 두 가지 경우를 설명하시오.
개념 ›
정답

Case 1: 크기 n의 인스턴스가 각각 거의 크기 n인 2개 이상의 인스턴스로 분할되는 경우
→ 실질적인 크기 감소가 없어서 재귀가 폭발적으로 증가

Case 2: 크기 n의 인스턴스가 거의 n개의 크기 n/c인 인스턴스로 분할되는 경우 (c: 상수)
→ 부분 문제의 수가 너무 많아 전체 작업량이 과도하게 증가

D
Knapsack DP에서 Θ(nW)인데 왜 이론적으로는 Θ(2ⁿ)이라고도 하는가?
개념 ›
정답

W는 입력 값이지, 입력 크기(input size)가 아님. W를 이진수로 표현하면 lg W 비트가 필요함.

즉, 비트 수 기준으로 보면 W = 2^(비트수) 이므로 Θ(nW) = Θ(n · 2^(lg W))가 되어 지수적 복잡도.

현실에서는 W가 크지 않아 잘 동작하지만, 이론적(worst case) 보장은 못 함.

이런 알고리즘을 pseudo-polynomial 알고리즘이라고 함!
