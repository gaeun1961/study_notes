Sequential Search (순차 탐색)

SeqSearch(int n, keyType[] S, keyType x) { index location = 1; while (location <= n && S[location] != x) // ← 빈칸! 비교 조건 location++; if (location > n) location = 0; // ← 빈칸! 못 찾은 경우 0 반환 return location; }

③ location=3, S[3]=11 = 11 → while 종료, return 3 ✓

Binary Search — 반복문 버전 (Ch.1)
정렬된 배열에서 절반씩 줄여가며 탐색
비유

전화번호부에서 'ㅎ' 찾기. 중간 펼치고 → 앞인지 뒤인지 결정 → 절반씩 버리기. 반드시 정렬된 배열 필요!

BinSearch(int n, keyType[] S, keyType x) { index location, low, high, mid; low = 1; high = n; location = 0; while (low <= high && location == 0) { mid = floor((low + high) / 2); if (x == S[mid]) location = mid; // ← 빈칸! else if (x < S[mid]) high = mid - 1; else low = mid + 1; // ← 빈칸! } return location; }

Basic Operation: x와 S[mid] 비교

W(n) = ⌊lg n⌋ + 1 ≈ Θ(lg n)
Fibonacci (재귀 vs 반복)
같은 문제를 두 가지 방식으로 풀면 효율이 완전히 달라져요

❌ 재귀 버전 (비효율)

Fib(int n) { if (n <= 1) return n; else return Fib(n-1) + Fib(n-2); // ← 빈칸! }

T(n) ≥ 2^(n/2) 지수적 증가

✅ 반복 버전 (효율)

Fib2(int n) { f[0] = 0; if (n > 0) { f[1] = 1; for (i=2; i<=n; i++) f[i] = f[i-1] + f[i-2]; // ← 빈칸! } return f[n]; }

T(n) = n+1 선형 시간

1.4 Order — 점근 표기법
n이 충분히 커지면 최고차항이 전체를 지배해요
핵심

0.1n² + n + 100 → n이 커지면 결국 Θ(n²)처럼 행동. 나머지 항은 무시!

표기조건의미
Big O∃c>0, ∃N≥0 s.t. g(n) ≤ c×f(n), ∀n≥N상한선
Omega Ω∃c>0, ∃N≥0 s.t. g(n) ≥ c×f(n), ∀n≥N하한선
Theta ΘO(f(n)) ∩ Ω(f(n))정확한 경계
small o∀c>0, ∃N≥0 s.t. g(n) ≤ c×f(n), ∀n≥N엄격한 상한 (모든 c에 대해)
Big O: ∃c 하나면 됨

n²+10n ∈ O(n²) → c=2, N=10 하나만 찾으면 OK

small o: 모든 c에 대해

n ∈ o(n²) → 어떤 c를 잡아도 결국 n ≤ cn² 성립

복잡도 순서

1 < lg n < n < n lg n < n² < n³ < 2ⁿ < n!

Order의 주요 성질

  • g(n) ∈ O(f(n)) ↔ f(n) ∈ Ω(g(n))
  • g(n) ∈ Θ(f(n)) ↔ f(n) ∈ Θ(g(n))
  • 로그 밑이 달라도 Θ는 같음: log_a(n) ∈ Θ(log_b(n))
  • b>a>0 이면 aⁿ ∈ o(bⁿ)
  • n!은 어떤 지수함수보다도 빠르게 증가: aⁿ ∈ o(n!)

Master Theorem
T(n) = aT(n/b) + f(n) 형태의 점화식을 빠르게 풀어요
Case조건결론
Case 1f(n) ∈ O(n^(log_b a − ε))T(n) = Θ(n^(log_b a))
Case 2f(n) ∈ Θ(n^(log_b a))T(n) = Θ(n^(log_b a) · lg n)
Case 3f(n) ∈ Ω(n^(log_b a + ε))T(n) = Θ(f(n))

2.2 MergeSort — mergeSort 코드
배열을 절반씩 나눠서 각각 정렬 후 합치기
비유

카드 더미를 절반씩 나눠서 각각 정렬한 뒤, 두 더미를 크기 순으로 합치기.

mergeSort(int n, keytype[] S) { if (n > 1) { const int h = floor(n/2), m = n - h; keytype[] U = new keytype[1..h], V = new keytype[1..m]; copy S[1:h] to U[1:h]; // 왼쪽 절반 복사 copy S[h+1:n] to V[1:m]; // ← 빈칸! 오른쪽 절반 mergeSort(h, U); mergeSort(m, V); // ← 빈칸! merge(h, m, U, V, S); } }
2.2 MergeSort — merge 코드
정렬된 두 배열을 하나로 합치는 핵심 함수
merge(int h, int m, keytype[] U, keytype[] V, keytype[] S) { index i=1, j=1, k=1; while (i <= h && j <= m) { if (U[i] < V[j]) { S[k] = U[i]; i++; } else { S[k] = V[j]; j++; } k++; } if (i > h) copy V[j:m] to S[k:h+m]; // U 소진 → 나머지 V 복사 else copy U[i:h] to S[k:h+m]; // ← 빈칸! V 소진 }

Basic Operation: U[i]와 V[j] 비교

Case공식
Best B(h,m)min(h,m)
Worst W(h,m)h+m-1
2.2 MergeSort — 시간복잡도
전체 MergeSort의 Worst Case 분석
W(n) = 2W(n/2) + n − 1   →   W(n) = n lg n − (n−1) ∈ Θ(n lg n)

실행 예시 — [27, 10, 12, 20, 25, 13]

① Divide: [27,10,12] | [20,25,13]

② 각각 재귀 정렬 → [10,12,27] | [13,20,25]

③ merge 결과: [10, 12, 13, 20, 25, 27]

2.4 QuickSort — quickSort 코드
pivot을 기준으로 작은 것은 왼쪽, 큰 것은 오른쪽. combine 불필요!
quickSort(index low, index high) { index pivotPoint; if (high > low) { pivotPoint = partition(low, high); quickSort(low, pivotPoint - 1); quickSort(pivotPoint + 1, high); // ← 빈칸! } }
2.4 QuickSort — partition 코드 (In-Place)
시험에서 수도코드 보고 In-Place인지 Local Array인지 구분해야 해요!
구분법
In-Place: S 배열 자체에서 swap, j는 low에서 시작해서 ++
Local Array: 별도 배열 L 사용
partition(index low, index high) { index i, j, pivotPoint; keytype pivotItem; pivotItem = S[low]; // 첫 원소를 pivot으로 j = low; for (i = low+1; i <= high; i++) if (S[i] < pivotItem) { // ← 빈칸! exchange S[i] and S[++j]; // 작은 원소를 왼쪽으로 } pivotPoint = j; exchange S[low] and S[pivotPoint]; // ← 빈칸! pivot 최종 위치로 return pivotPoint; }

Basic Operation: S[i]와 pivotItem 비교  |  T(m) = m−1 (Every Case)

partition 실행 예시
S=[15, 22, 13, 27, 12, 10, 20, 25]에서 partition(1,8)

pivot = S[1] = 15, j=1

i=2: 22 ≥ 15 → skip

i=3: 13 < 15 → j=2, swap(S[3],S[2]) → [15,13,22,27,12,10,20,25]

i=5: 12 < 15 → j=3, swap → [15,13,12,27,22,10,20,25]

i=6: 10 < 15 → j=4, swap → [15,13,12,10,22,27,20,25]

마지막: swap(S[1],S[4]) → [10,13,12,15,22,27,20,25]

→ pivotPoint=4. 15 왼쪽 모두 <15, 오른쪽 모두 ≥15 ✓

3.1 Binomial Coefficient (이항계수)
C(n,k) = n개 중 k개 고르는 경우의 수
C(n,k) = C(n-1,k-1) + C(n-1,k)  (0<k<n)  |  C(n,0) = C(n,n) = 1
bin2(int n, int k) { for i=0 to n: for j=0 to min(i,k): if (j==0 || j==i) B[i][j] = 1; else B[i][j] = B[i-1][j-1] + B[i-1][j]; return B[n][k]; }

T(n,k) ∈ Θ(nk)

3.2 Floyd's Algorithm (최단 경로)
모든 쌍(pair) 간 최단 경로를 한 번에 구해요
비유

서울→부산 갈 때, 대구를 거치는 게 더 빠른지 확인. 모든 중간 도시를 하나씩 추가하면서 최단 경로 업데이트.

D(k)[i][j] 의미

{v1,...,vk}만 경유 허용 시 vi→vj 최단 거리. D(0)[i][j] = W[i][j]

D(k)[i][j] = min( D(k-1)[i][j] , D(k-1)[i][k] + D(k-1)[k][j] )
floyd(int n, number W[][], number D[][]) { D = W; for k=1 to n: for i=1 to n: for j=1 to n: D[i][j] = min(D[i][j], D[i][k] + D[k][j]); }

T(n) = n³  |  경로 복원: P[i][j]에 경유 노드 k 저장

3.4 Optimal Binary Search Tree
자주 검색하는 키를 위쪽에 배치해서 전체 탐색 비용 최소화
A[i][j] 의미

Keyi ~ Keyj로 만든 최적 BST의 최소 ΣPl×Cl

A[i][j] = min(i≤k≤j) { A[i][k-1] + A[k+1][j] } + Σ(m=i~j) Pm

기저 조건

  • A[i][i] = Pi (키 1개짜리)
  • A[i][i-1] = 0, A[j+1][j] = 0 (빈 트리 = 0)

테이블 범위 & 채우기 순서

  • 행: 1 ~ n+1   열: 0 ~ n (빈 트리 케이스 저장용)
  • 채우는 순서: 대각선 방향 (키 1개 → 2개 → ... → n개)
왜 n+1까지? 점화식에서 A[n+1][n] 같은 빈 트리 케이스가 등장하기 때문!

T(n) ∈ Θ(n³)  |  R[i][j]: 최솟값 달성한 k → 트리 복원에 사용

Optimal BST — A값 계산 예시
P1=0.1, P2=0.3, P3=0.6일 때 단계별 계산

기저값
A[1][1]=0.1, A[2][2]=0.3, A[3][3]=0.6
A[i][i-1]=0 (빈 트리들)

A[1][2] (P1+P2=0.4)

k=1: A[1][0]+A[2][2]+0.4 = 0+0.3+0.4 = 0.7

k=2: A[1][1]+A[3][2]+0.4 = 0.1+0+0.4 = 0.5 ← min

A[2][3] (P2+P3=0.9)

k=2: 0+0.6+0.9 = 1.5   k=3: 0.3+0+0.9 = 1.2 ← min

A[1][3] (P1+P2+P3=1.0)

k=1: 0+1.2+1.0=2.2   k=2: 0.1+0.6+1.0=1.7   k=3: 0.5+0+1.0=1.5 ← min

3.6 TSP — 외판원 문제
모든 도시를 한 번씩 방문하고 돌아오는 최단 경로
D[vi][A] 의미

A의 모든 정점을 경유하여 vi → v1 최단 경로 길이
D[vi][∅] = W[i][1]

D[vi][A] = min(j: vj∈A) { W[i][j] + D[vj][A−{vj}] }

최적 투어 길이

min(2≤j≤n) { W[1][j] + D[vj][V−{v1,vj}] }

계산 순서: A 크기 0 → 1 → ... → n-2

T(n) ∈ Θ(n² · 2ⁿ)

더 좋은 알고리즘은 아직 없음 (NP-hard). Factorial에서 n²·2ⁿ으로 줄인 것.
4.5.4 Knapsack — 문제 정의
무게 제한이 있는 배낭에 이익을 최대화하도록 물건 선택
비유

값비싼 것만 고르면 무거울 수 있고, 가벼운 것만 고르면 이익이 적을 수 있음 → 최적 조합 찾기!

  • n개 물건, 각각 무게 wi, 이익 pi
  • 배낭 최대 무게 W
  • 목표: Σwi ≤ W 조건에서 Σpi 최대화
  • 모든 값은 양의 정수
Knapsack — 점화식
P[i][w]의 의미와 두 가지 경우로 나누는 핵심 아이디어
P[i][w] 의미

처음 i개 물건만 사용 가능하고, 무게 제한이 w일 때 달성 가능한 최대 이익

wi > w 이면: P[i][w] = P[i-1][w]

wi ≤ w 이면: P[i][w] = max( P[i-1][w] , pi + P[i-1][w-wi] )

P[i-1][w]: i번 물건 안 넣는 경우

pi + P[i-1][w-wi]: i번 물건 넣는 경우

기저 조건

  • P[0][w] = 0   P[i][0] = 0

최종 답: P[n][W]

Knapsack — 테이블 예시
n=3, W=5, (w1=2,p1=3), (w2=3,p2=4), (w3=4,p3=5)
w=0w=1w=2w=3w=4w=5
i=0000000
i=1 (w=2,p=3)003333
i=2 (w=3,p=4)003447
i=3 (w=4,p=5)003457
P[3][5] = 7. 물건1(무게2,이익3) + 물건2(무게3,이익4) = 무게5, 이익7 ✓
Knapsack — 시간복잡도
이중 for문 → Θ(nW)
for i=1 to n: // n번 for w=1 to W: // W번 P[i][w] = ... // O(1)
T(n, W) = Θ(nW)
이론적으로 n만으로 보면 Θ(2ⁿ). W가 매우 크면 느릴 수 있어 이론상 보장은 못함.
현실적으로는 W가 크지 않아서 잘 동작!

문제 원문

We have already studied a balanced two-way mergesort in the lecture. Now (1) give the pseudo-code of a balanced three-way mergesort and (2) compute its worst-case time complexity.

한국어 해석

강의에서 이미 균형 잡힌 이분할 병합정렬을 배웠습니다.
이제 (1) 균형 잡힌 삼분할 병합정렬의 수도코드를 작성하고, (2) 최악의 경우 시간복잡도를 계산하시오.

풀이 아이디어

달라지는 것은 딱 두 가지!
① 2등분 → 3등분
② merge(두 배열) → merge3(세 배열)

과제 #1-1: Three-way MergeSort — 수도코드
교수님 Two-way 스타일을 그대로 확장
mergesort3(int n, keytype[] S) { if (n > 1) { const int h = floor(n/3), m = floor(n/3), p = n-h-m; keytype[] U = new keytype[1..h], V = new keytype[1..m], W = new keytype[1..p]; copy S[1:h] to U[1:h]; copy S[h+1:h+m] to V[1:m]; copy S[h+m+1:n] to W[1:p]; mergesort3(h, U); mergesort3(m, V); mergesort3(p, W); merge3(h, m, p, U, V, W, S); } } merge3(int p, int q, int r, keytype[] U, keytype[] V, keytype[] W, keytype[] S) { index i=1, j=1, k=1, l=1; // 셋 다 남아있는 동안 — 세 원소 중 최솟값 선택 while (i<=p && j<=q && k<=r) { if (U[i] <= V[j] && U[i] <= W[k]) { S[l]=U[i]; i++; } else if (V[j] <= U[i] && V[j] <= W[k]) { S[l]=V[j]; j++; } else { S[l]=W[k]; k++; } l++; } // 두 개만 남은 경우 — two-way merge while (i<=p && j<=q) { if (U[i]<=V[j]) { S[l]=U[i]; i++; } else { S[l]=V[j]; j++; } l++; } while (i<=p && k<=r) { if (U[i]<=W[k]) { S[l]=U[i]; i++; } else { S[l]=W[k]; k++; } l++; } while (j<=q && k<=r) { if (V[j]<=W[k]) { S[l]=V[j]; j++; } else { S[l]=W[k]; k++; } l++; } // 하나만 남은 경우 — 그대로 복사 if (i<=p) copy U[i:p] to S[l:l+p-i]; if (j<=q) copy V[j:q] to S[l:l+q-j]; if (k<=r) copy W[k:r] to S[l:l+r-k]; }
과제 #1-1: Three-way MergeSort — 시간복잡도
Master Theorem으로 빠르게 풀기
점화식

T(n) = 3T(n/3) + n   (3번 재귀 + merge3에서 n번 비교)

Master Theorem: a=3, b=3, log₃3=1, f(n)=n=Θ(n¹) → Case 2

T(n) = Θ(n log n)
Two-way MergeSort와 시간복잡도가 같아요! (상수 계수만 다름)
과제 #1-2: 동전 거스름돈 — 문제
DP로 경우의 수를 구하는 문제
문제 원문

Given a value N, if we want to make change for N cents, and we have infinite supply of each of S = {S1, S2, ..., Sm} valued coins, how many ways can we make the change? The order of coins does not matter.

For N=4 and S={1,2,3}, there are four ways: {1,1,1,1}, {1,1,2}, {2,2}, {1,3}. Output = 4.

한국어 해석

N센트를 S={S1,...,Sm} 동전으로 만들 수 있는 경우의 수를 구하라. (순서 무관, 무한 공급)

(1) 알고리즘 아이디어를 자세히 설명하고
(2) 수도코드를 작성하고
(3) 시간복잡도를 분석하시오.

과제 #1-2: 동전 거스름돈 — 아이디어
DP 테이블 C[i][j]의 의미와 점화식 유도
C[i][j] 의미

동전 S1~Si만 사용해서 j원을 만드는 경우의 수

핵심 아이디어

Si를 한 번도 안 쓰는 경우 → C[i-1][j]
Si를 최소 1번 쓰는 경우 → C[i][j-Si] (Si 또 쓸 수 있으니 i 유지!)

두 경우를 더하면 전체 경우의 수!

j < Si 이면: C[i][j] = C[i-1][j]

j ≥ Si 이면: C[i][j] = C[i-1][j] + C[i][j-Si]

기저 조건

  • C[i][0] = 1 (0원 만드는 방법: 아무것도 안 쓰는 1가지)
  • C[0][j] = 0 (동전 없으면 0가지)

최종 답: C[m][N]

과제 #1-2: 동전 거스름돈 — 수도코드
점화식을 그대로 코드로 옮기면 돼요
coinChange(int N, int S[], int m) { for i=0 to m: C[i][0] = 1; // 0원 만드는 방법: 1가지 for j=1 to N: C[0][j] = 0; // 동전 없으면 0가지 for i=1 to m: for j=1 to N: C[i][j] = C[i-1][j]; if (j >= S[i]) C[i][j] += C[i][j - S[i]]; return C[m][N]; }

검증 — N=4, S={1,2,3}

j=0j=1j=2j=3j=4
i=010000
i=1 (S1=1)11111
i=2 (S2=2)11223
i=3 (S3=3)11234
C[3][4] = 4 ✓
과제 #1-2: 동전 거스름돈 — 시간복잡도
이중 for문 → Θ(mN)
for i=1 to m: // m번 반복 for j=1 to N: // N번 반복 O(1) // 각 칸은 상수 시간

이중 for문이므로

T(m, N) = Θ(mN)
