1.1 인간 시각의 강점과 한계 ★
PPT 3~5p

시각은 오감 중에서 가장 뛰어나요. 매 순간 빠르고 정확하게, 아주 손쉽게 인식해요.

인간 시각의 강점
  • 분류, 검출, 분할, 추적, 행동 분석에 능숙
  • 3차원 복원 능력
  • 빠르고 강건(robust)
  • 지식 표현, 추론, 계획과 협동
  • 사전 행동(proactive)에 능숙
  • 과업 전환이 매끄럽고 유기적이고 빠름
  • 비주얼 서보잉이 뛰어남
인간 시각의 한계
  • 착시가 있음
  • 정밀 측정에 오차
  • 시야가 한정됨
  • 피로해지고 퇴화
1.3 컴퓨터 비전은 왜 어려운가? ★
PPT 11p — 넘버 크런처!

컴퓨터 비전이 어려운 이유는 명확해요. 크게 세 가지예요.

① 세상의 변화무쌍함
  • 환경(낮밤, 날씨 등) 변화, 보는 위치와 방향의 변화, 강체와 연성 물체
  • 원자부터 우주까지 긴 스펙트럼에서 영상 수집
② 컴퓨터는 넘버 크런처

컴퓨터는 숫자 계산기예요. 이미지도 결국 숫자(픽셀)의 배열일 뿐이에요. 사람은 "강아지"를 보면 바로 인식하지만, 컴퓨터에게는 0~255 숫자 배열에 불과해요. 의미(meaning)를 이해하지 못하고 계산만 해요.

③ 인공지능의 미숙함
  • 지식 표현, 추론, 계획, 학습이 유기적으로 동작할 때만 강한 인공지능 가능
  • 강한 인공지능은 먼 미래의 일 또는 영영 불가능
⭐ 핵심: 컴퓨터 비전이 어려운 이유 = 세상의 변화무쌍함 + 넘버 크런처 + 인공지능의 미숙함
1.6 컴퓨터 비전의 목표
PPT 22p

일반적인 상황에서 잘 작동하는 인간과 같은 시각 (강한 인공지능)

→ 컴퓨터 비전 문제를 여러 세부 문제로 구분하고 세부 문제별로 알고리즘 구상

컴퓨터 비전은 인간의 시각을 흉내 내는 컴퓨터 프로그램. 현재 기술로 인간에 필적하는 시각 구현은 불가능하지만, 과업을 한정하면 인간 성능에 가깝거나 뛰어넘는 응용이 무궁무진해요!
이미지 구조
컴퓨터가 이미지를 저장하는 방식

이미지는 숫자들의 격자판이에요. 각 픽셀에 0~255 숫자가 저장돼요.

0
완전 검정
128
중간 회색
255
완전 흰색

컬러 이미지 = BGR 3채널

OpenCV는 BGR 순서로 저장해요 (일반 RGB와 반대!).

OpenCV = B(0번) + G(1번) + R(2번). 빨간 채널이 2번!

⚠️ OpenCV는 BGR 순서! 빨간 채널 → img[:,:,2]

OpenCV는 numpy.ndarray로 영상 표현 — numpy 함수(min, max, mean, reshape 등) 바로 사용 가능!

3.1.1 디지털 변환 — 샘플링 vs 양자화 ★
PPT 5~6p — 객관식 출제!
샘플링 (Sampling)
  • 연속된 데이터 중 유한한 개수의 데이터를 뽑아내는 행위
  • 해상도와 관련 있는 개념
  • 연속 신호에서 일정 간격으로 샘플 추출 → 픽셀 단위로 나누는 과정
  • M×N 영상으로 샘플링
샘플 많을수록 → 해상도 높아짐
양자화 (Quantization)
  • 아날로그 형태로 샘플링한 데이터를 디지털화
  • 비트(bit)수와 관련 있는 개념
  • 픽셀의 색상 값을 8bit의 0~255 사이 정수로 변환
  • L단계로 양자화 (8bit = L=256단계)
비트수 높을수록 → 더 자연스러운 색
⭐ 핵심: 샘플링=픽셀 개수(해상도) / 양자화=픽셀 밝기값(비트수)
M×N 영상으로 샘플링하고 L단계로 양자화!

3.2.2 이진화 · Otsu 알고리즘 ★
PPT 18~19p — 반환값 2개 주의!

이진화: 픽셀을 0(검정) 또는 255(흰색) 둘 중 하나로만 만들기.

Otsu: 이진화를 최적화 문제로 바라봄. 임계값 T를 자동으로 찾아줘요.

t̂ = argmin J(t)    J(t) = n₀(t)v₀(t) + n₁(t)v₁(t)     (3.2, 3.3)

목적함수 J(t): 작을수록 좋음. 0이 되는 화소들의 분산과 1이 되는 화소들의 분산의 가중치 합

반환값이 2개! t(임계값 숫자) + bin_img(결과 이미지). 순서 틀리면 오류!
print(t)로 Otsu가 자동 계산한 임계값 확인 — 이 줄도 시험에 나와요!
3.2.4 모폴로지 ★
PPT 22p — 객관식!

구조 요소(structuring element)를 이용하여 영역의 모양을 조작해요.

팽창 (Dilation)

작은 홈 메우기, 끊어진 영역 연결. 영역 키움

침식 (Erosion)

경계 돌출 부분 깎기. 영역 작게

열림 (Opening) = 침식→팽창

작은 노이즈 제거. 원래 크기 유지

닫힘 (Closing) = 팽창→침식

작은 구멍 메우기. 원래 크기 유지

열림/닫힘 모두 원래 크기로 돌아와요. 열림은 침식 먼저(외부 돌기 제거), 닫힘은 팽창 먼저(내부 구멍 메우기)!

gray16=np.int16(gray) # 컨볼루션할 때 int16 변환 (음수 저장 위해) emboss=np.uint8(np.clip(cv.filter2D(gray16,-1,femboss)+128,0,255)) # ① cv.filter2D: 컨볼루션 # ② +128: 음수를 양수 구간으로 올리기 (각 화소에 +128) # ③ np.clip: 0~255로 강제 조정 (오버플로우 방지) # ④ np.uint8: 최종 변환

주변 4개 픽셀을 넓이 비율(α, β)로 가중합산. 더 자연스럽고 부드러움!

양선형 보간 수식 ★계산문제

α = i' − i (x 소수 부분), β = j' − j (y 소수 부분)

f ≈ αβ·f(j,i) + (1-α)β·f(j,i+1) + α(1-β)·f(j+1,i) + (1-α)(1-β)·f(j+1,i+1)
직관: 빨간 점이 f(j,i)에 가까울수록 그 픽셀값이 더 많이 반영돼요!

4.1 에지(Edge)란? · 미분
PPT 3~9p
에지(Edge)

물체 경계에 있는 점. 내부는 명암이 서서히 변하고 경계는 급격히 변하는 특성 활용

f'(x) ≈ f(x+1) - f(x)    (전진 차분)

현실 세계의 램프 에지 — 두꺼운 에지 문제

이상적 에지는 1픽셀이지만 현실에서는 명암이 몇 화소에 걸쳐 변함 → 두꺼운 에지 발생 → 위치 찾기 문제!

1차 미분

에지에서 봉우리(극대값)가 나타남

2차 미분

에지에서 영교차(zero-crossing)가 나타남

⭐ 에지 검출: 1차 미분에서 봉우리 찾기 / 2차 미분에서 영교차 찾기

5.2.1 모라벡 알고리즘 ★계산
PPT 9~11p — S(0,1)=4 계산!
S(v,u) = ΣΣ (f(y+v, x+u) − f(y,x))²     (5.1)
특징점 가능성 계산
  • 상하좌우 4방향으로 각각 S값 계산
  • 특징점 가능성 = min(S 상, S 하, S 좌, S 우)
  • min: 가장 약한 방향도 커야 진짜 코너!
S(0,1) = Σ Σ (f(y, x+1) − f(y,x))² = 4     (예시 5-1)
평탄

S 모두 작음 → min 작음

에지

한 방향만 큼 → min 작음

코너 ★

모두 큼 → min 큼 = 특징점!

import cv2 as cv import numpy as np img=cv.imread('mot_color70.jpg') gray=cv.cvtColor(img,cv.COLOR_BGR2GRAY) harris=cv.cornerHarris(gray,blockSize=4,ksize=3,k=0.04) # C값 맵 반환 harris=cv.dilate(harris,None) # 팽창으로 NMS(극대점 찾기) img[harris>0.01*harris.max()]=[0,0,255] # 빨간점으로 표시 cv.imshow('Harris corners',img) cv.waitKey(); cv.destroyAllWindows()

import cv2 as cv import numpy as np import time img1=cv.imread('mot_color70.jpg')[190:350,440:560] # 버스 크롭 → 모델 영상 gray1=cv.cvtColor(img1,cv.COLOR_BGR2GRAY) img2=cv.imread('mot_color83.jpg') # 장면 영상 gray2=cv.cvtColor(img2,cv.COLOR_BGR2GRAY) sift=cv.SIFT_create() kp1,des1=sift.detectAndCompute(gray1,None) kp2,des2=sift.detectAndCompute(gray2,None) print('특징점 개수:',len(kp1),len(kp2)) # ① flann_params=dict(algorithm=1,trees=5) search_params=dict(checks=50) flann=cv.FlannBasedMatcher(flann_params,search_params) start=time.time() matches=flann.knnMatch(des1,des2,k=2) # k=2: 최근접 2개 찾기 print('매칭에 걸린 시간:',time.time()-start) # ② 아주 빠른 FLANN! good_matches=[] for m,n in matches: if m.distance<0.7*n.distance: # Lowe's ratio test: 70%이하면 채택 good_matches.append(m) res=cv.drawMatches(img1,kp1,img2,kp2,good_matches,None) cv.imshow('Good Matches',res) cv.waitKey(); cv.destroyAllWindows()
