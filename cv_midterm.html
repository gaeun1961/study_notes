인간 시각의 강점
  • 분류, 검출, 분할, 추적, 행동 분석에 능숙

  • 비주얼 서보잉이 뛰어남

  • 착시가 있음
  • 정밀 측정에 오차
  • 시야가 한정됨
  • 피로해지고 퇴화
1.3 컴퓨터 비전은 왜 어려운가? ★
PPT 11p — 넘버 크런처!

컴퓨터 비전이 어려운 이유는 명확해요. 크게 세 가지예요.

① 세상의 변화무쌍함
  • 환경(낮밤, 날씨 등) 변화, 보는 위치와 방향의 변화, 강체와 연성 물체
  • 원자부터 우주까지 긴 스펙트럼에서 영상 수집
② 컴퓨터는 넘버 크런처

컴퓨터는 숫자 계산기예요. 이미지도 결국 숫자(픽셀)의 배열일 뿐이에요. 사람은 "강아지"를 보면 바로 인식하지만, 컴퓨터에게는 0~255 숫자 배열에 불과해요. 의미(meaning)를 이해하지 못하고 계산만 해요.

③ 인공지능의 미숙함
  • 지식 표현, 추론, 계획, 학습이 유기적으로 동작할 때만 강한 인공지능 가능
  • 강한 인공지능은 먼 미래의 일 또는 영영 불가능
⭐ 핵심: 컴퓨터 비전이 어려운 이유 = 세상의 변화무쌍함 + 넘버 크런처 + 인공지능의 미숙함

컬러 이미지 = BGR 3채널

OpenCV는 BGR 순서로 저장해요 (일반 RGB와 반대!).

OpenCV = B(0번) + G(1번) + R(2번). 빨간 채널이 2번!

⚠️ OpenCV는 BGR 순서! 빨간 채널 → img[:,:,2]

OpenCV는 numpy.ndarray로 영상 표현 — numpy 함수(min, max, mean, reshape 등) 바로 사용 가능!

3.1.1 디지털 변환 — 샘플링 vs 양자화 ★
PPT 5~6p — 객관식 출제!
샘플링 (Sampling)
  • 연속된 데이터 중 유한한 개수의 데이터를 뽑아내는 행위
  • 해상도와 관련 있는 개념
  • 연속 신호에서 일정 간격으로 샘플 추출 → 픽셀 단위로 나누는 과정
  • M×N 영상으로 샘플링
샘플 많을수록 → 해상도 높아짐
양자화 (Quantization)
  • 아날로그 형태로 샘플링한 데이터를 디지털화
  • 비트(bit)수와 관련 있는 개념
  • 픽셀의 색상 값을 8bit의 0~255 사이 정수로 변환
  • L단계로 양자화 (8bit = L=256단계)
비트수 높을수록 → 더 자연스러운 색
⭐ 핵심: 샘플링=픽셀 개수(해상도) / 양자화=픽셀 밝기값(비트수)
M×N 영상으로 샘플링하고 L단계로 양자화!

3.2.4 모폴로지 ★
PPT 22p — 객관식!

구조 요소(structuring element)를 이용하여 영역의 모양을 조작해요.

팽창 (Dilation)

작은 홈 메우기, 끊어진 영역 연결. 영역 키움

침식 (Erosion)

경계 돌출 부분 깎기. 영역 작게

열림 (Opening) = 침식→팽창

작은 노이즈 제거. 원래 크기 유지

닫힘 (Closing) = 팽창→침식

작은 구멍 메우기. 원래 크기 유지

열림/닫힘 모두 원래 크기로 돌아와요. 열림은 침식 먼저(외부 돌기 제거), 닫힘은 팽창 먼저(내부 구멍 메우기)!

컨볼루션이란?

각 화소에 필터 u를 적용해 주변 픽셀과 필터를 곱하고 더해 새로운 픽셀 값을 만드는 영역 기반 연산이에요.

1차원: f'(x) = Σ u(i)·f(x+i)    2차원: f'(y,x) = ΣΣ u(j,i)·f(y+j, x+i)

컨볼루션 계산 예시 (PPT 36p)

수직 에지를 검출하는 필터 [-1, 1]을 영상에 적용하면:

위치 3의 계산: f'(3) = (-1)×f(3) + (1)×f(4) = (-1)×5 + (1)×7 = 2

핵심: 필터를 밀면서 각 위치에서 필터값과 영상값을 곱해서 더하면 그게 새 픽셀값! 에지 쪽에서 큰 값이 나와요.

3.5 영상 보간법 ★★★
PPT 50~52p — 양선형 보간법 계산문제 무조건!

왜 보간이 필요한가?

영상을 기하 변환(회전, 확대 등)하면 결과 픽셀 좌표가 실수(소수점)로 나와요. 예를 들어 2.7배 확대하면 좌표 1이 → 2.7이 되는데 이걸 어떤 픽셀값으로 채울지 결정하는 게 보간이에요.

최근접 이웃 방법
  • 실수 좌표를 반올림해서 가장 가까운 픽셀값을 그대로 사용
  • 계산이 빠르고 단순
  • 에일리어싱(계단 현상) 발생 — 픽셀이 뭉개져 보임
예) 좌표 2.7 → round(2.7) = 3번 픽셀값 사용
양선형 보간법 ★★
  • 걸치는 비율(α, β)에 따라 주변 4개 픽셀을 가중합산
  • 안티 에일리어싱 — 부드럽고 자연스러운 결과
  • 최근접보다 계산 비용이 약간 더 큼
예) 좌표 2.7 → 주변 픽셀 2, 3 사이를 0.3:0.7 비율로 섞음

양선형 보간 수식 ★계산문제

실수 좌표 (j', i')가 정수 픽셀 (j, i) 사이에 걸쳐 있을 때:

α = i' − i    (x방향 소수 부분, 오른쪽으로 얼마나 걸쳤는지)
β = j' − j    (y방향 소수 부분, 아래로 얼마나 걸쳤는지)
f(j',i') ≈ (1-α)(1-β)·f(j,i) + α(1-β)·f(j,i+1) + (1-α)β·f(j+1,i) + αβ·f(j+1,i+1)

P가 f(j,i)에 가까울수록(α,β 작을수록) f(j,i) 값이 더 많이 반영!

계산 예시

실수 좌표 (j', i') = (1.3, 0.7) → j=1, i=0, α=0.7, β=0.3

주변 4픽셀: f(1,0)=10, f(1,1)=20, f(2,0)=30, f(2,1)=40

f(1.3, 0.7) ≈ (1-0.7)(1-0.3)×10 + 0.7(1-0.3)×20 + (1-0.7)×0.3×30 + 0.7×0.3×40

= 0.21×10 + 0.49×20 + 0.09×30 + 0.21×40 = 2.1 + 9.8 + 2.7 + 8.4 = 23

5.2.1 모라벡 알고리즘 ★계산
PPT 9~11p — S(0,1)=4 계산!

모라벡의 아이디어

a(코너)는 여러 방향으로 색상 변화가 있어 찾기 쉬운데 c(평탄)는 어느 방향으로도 미세한 변화만 있어 찾기 어려워요. 이를 수식으로 표현한 것이 제곱차의 합 S(v,u)예요.

S(v,u) = ΣΣ (f(y+v, x+u) − f(y,x))²     (5.1)

이동 방향(v,u), 움직인 후 밝기 f(y+v,x+u), 원래 밝기 f(y,x)

식 (5.1)의 의도

c (평탄)

모든 방향에서 변화 없음
→ S값 모두 0
→ 특징 자격 없음

b (에지)

수평 방향만 변화
→ S값 좌우 이웃만 큼
→ 특징으로 부족

a (코너) ★

모든 방향으로 변화
→ S값 모두 큼
→ 훌륭한 특징점!

특징점 가능성 값 측정

계산 방법
  • 상하좌우 4방향 각각 S값 계산 → S 맵 생성
  • 특징 가능성 = S 맵에서 상하좌우 네 이웃 화소의 최솟값
  • min을 쓰는 이유: 가장 약한 방향도 커야 진짜 코너!
  • 이 값이 임계값보다 크면 특징점

[예시 5-1] S(0,1) 계산 ★직접계산

화소 위치 (y,x) = (4,3)에서 오른쪽으로 1칸 이동한 경우 (v=0, u=1):

S(0,1) = Σ Σ (f(y, x+1) − f(y,x))² = 4
시험 팁: (v,u) = (0,1)은 x방향(오른쪽), (1,0)은 y방향(아래), (-1,0)은 위, (0,-1)은 왼쪽!

전략방법특징
고정 임계값 거리 d(ai,bj) < T인 모든 쌍을 매칭 T 설정이 어려움
최근접 이웃 ai마다 가장 가까운 bj를 찾아 d < T이면 매칭 조금 더 나음
최근접 이웃 거리 비율 (Lowe's ratio) 1번째 가장 가까운 bj와 2번째 가까운 bk의 거리 비율 < T이면 매칭 세 가지 중 가장 높은 성능!
⭐ Lowe's ratio test: m.distance < 0.7 * n.distance → 첫번째가 두번째보다 충분히 가까울 때만 채택!

📝 계산 연습 — 모라벡 min값 ★★★기출!
오전반 기출: 모라벡 알고리즘으로 min값 구하기

계산 순서

⭐ ① 4방향 S값 계산 → ② min 구하기
S(v,u) = Σ(f(y+v, x+u) − f(y,x))²    ← 차이의 제곱합!
방향(v,u)의미
오른쪽(0, +1)x만 +1 이동
왼쪽(0, -1)x만 -1 이동
아래(+1, 0)y만 +1 이동
위(-1, 0)y만 -1 이동

예시 1 — 1차원 영상에서 S(0,1) 계산

영상 (1행): x: 0 1 2 3 4 5 값: 3 3 3 7 7 7 S(0,1) 계산 — 오른쪽으로 한 칸 이동: (f(0,1)−f(0,0))² = (3−3)² = 0 (f(0,2)−f(0,1))² = (3−3)² = 0 (f(0,3)−f(0,2))² = (7−3)² = 16 ← 에지 위치! (f(0,4)−f(0,3))² = (7−7)² = 0 (f(0,5)−f(0,4))² = (7−7)² = 0 S(0,1) = 0+0+16+0+0 = 16

예시 2 — 2차원 영상에서 4방향 min값 구하기

영상: x=0 x=1 x=2 x=3 x=4 y=0 [ 3 3 3 3 3 ] y=1 [ 3 3 3 3 3 ] y=2 [ 3 3 3 7 7 ] y=3 [ 3 3 7 7 7 ] y=4 [ 3 7 7 7 7 ] 중심 화소: (y=3, x=2) → f(3,2) = 7 S(0,+1) — 오른쪽: (f(3,3)−f(3,2))² = (7−7)² = 0 ← 주변 한 칸만 보는 경우 S(0,−1) — 왼쪽: (f(3,1)−f(3,2))² = (3−7)² = 16 S(+1,0) — 아래: (f(4,2)−f(3,2))² = (7−7)² = 0 S(−1,0) — 위: (f(2,2)−f(3,2))² = (3−7)² = 16 특징점 가능성 = min(0, 16, 0, 16) = 0 → 오른쪽/아래 방향 변화 없음 → 코너 아님!

⚠️ min이 0이면 특징점이 아니에요! 한 방향이라도 변화 없으면 탈락!

진짜 코너라면?
영상: 3 3 3 3 7 7 ← 중심 화소 = 7 (y=1, x=1) 3 7 7 S(0,+1): (7−7)² = 0 ← 오른쪽 같음 S(0,−1): (3−7)² = 16 ← 왼쪽 다름 S(+1,0): (7−7)² = 0 ← 아래 같음 S(−1,0): (3−7)² = 16 ← 위 다름 min(0, 16, 0, 16) = 0 → 여전히 0! 오른쪽/아래가 같아서 코너 아님. 진짜 코너: 3 3 3 3 7 3 ← 상하좌우 모두 다른 경우 3 3 3 S 상하좌우 모두 큼 → min 큼 → 특징점! ★
