이미지 구조
컴퓨터가 이미지를 저장하는 방식

우리 눈에는 사진처럼 보이지만 컴퓨터한테 이미지는 그냥 숫자들의 격자판이에요. 각 칸(픽셀)에 밝기를 나타내는 0~255 숫자가 저장돼요.

이미지 = 숫자 격자판. 각 픽셀에 0~255 숫자 저장

0
완전 검정
128
중간 회색
255
완전 흰색

컬러 이미지 = BGR 3채널

컬러 이미지는 파랑(B)·초록(G)·빨강(R) 3장의 흑백 이미지가 겹쳐진 거예요. OpenCV는 BGR 순서로 저장해요!

B(0번) + G(1번) + R(2번). R이 2번인 거 기억!

⚠️ OpenCV는 BGR 순서! 일반 RGB와 반대예요. 빨간 채널을 꺼내려면 img[:,:,2]
좌표계·채널 접근
이미지 배열 접근 방법

수학에서는 y축이 위로 올라가지만, 이미지에서는 y축이 아래로 내려가요. 원점(0,0)은 왼쪽 위 모서리예요.

원점(0,0)은 왼쪽 위. y가 아래로 증가. 배열 접근: img[y, x, 채널]

y(세로)가 먼저, x(가로)가 나중! img[y, x, ch] 순서예요.

디지털 변환
샘플링 vs 양자화 — 객관식 출제!

현실의 연속적인 장면을 컴퓨터 숫자로 바꾸는 2단계 과정이에요.

현실 → 샘플링 → 양자화 → 디지털 이미지

샘플링 (Sampling)

연속적인 장면을 격자로 나눠서 픽셀을 추출. 카메라 해상도를 결정해요.

비유: 그림을 모눈종이에 옮겨 그리는 것. 칸이 많을수록 더 세밀!

샘플 많을수록 → 해상도 높아짐
양자화 (Quantization)

각 픽셀의 밝기를 0~255 정수로 변환. 비트수와 관련있어요.

비유: 무한한 색을 256가지로 줄이는 것. 8bit = 256단계!

비트수 많을수록 → 더 자연스러운 색
핵심 구별법: 샘플링 = 픽셀 개수(해상도), 양자화 = 픽셀 밝기값(비트수)
이진화와 Otsu 알고리즘
cv.threshold + THRESH_OTSU ★반환값 2개!

이진화란 이미지 픽셀을 0(검정) 또는 255(흰색) 둘 중 하나로만 만드는 거예요. Otsu는 임계값 T를 자동으로 찾아줘요.

임계값 T를 기준으로 밝으면 255, 어두우면 0으로 이분화

# Otsu 이진화 — 반환값이 2개! t, bin_img = cv.threshold(img[:,:,2], 0, 255, cv.THRESH_BINARY + cv.THRESH_OTSU) # ↑R채널 ↑0은 무의미(Otsu가 자동 계산) # ↑OTSU: 자동으로 T 찾기 print('자동 임계값:', t) # t = 자동 계산된 임계값 숫자
반환값이 2개예요! t (임계값 숫자) 와 bin_img (결과 이미지). 순서 틀리면 오류!
print(t)로 Otsu가 자동 계산한 임계값 확인 가능 — 이 줄도 시험에 나와요!
인자/반환값의미
thresh0Otsu 쓸 땐 무의미 (자동 계산됨)
maxval255T 이상일 때 채울 값
typeTHRESH_BINARY+THRESH_OTSU이진화 방식 + 자동 임계값
반환① t숫자자동으로 찾은 최적 임계값
반환② bin_img이미지이진화된 결과 이미지
모폴로지 연산
팽창·침식·열림·닫힘 — 객관식 출제!

이진 이미지의 형태를 조작하는 연산이에요. 구조 요소(kernel)를 이미지 위에 슬라이딩하며 적용해요.

팽창 (Dilation)

구조 요소를 씌웠을 때 하나라도 1이면 중심 픽셀을 1로 만들어요.

효과: 흰색 영역 커짐, 끊어진 선 연결

b_dilation = cv.dilate(b, se, iterations=1)
침식 (Erosion)

구조 요소를 씌웠을 때 모두 1이어야만 중심 픽셀을 1로 유지해요.

효과: 흰색 영역 줄어듦, 노이즈 제거

b_erosion = cv.erode(b, se, iterations=1)

팽창은 흰 영역이 커지고, 침식은 줄어들어요

열림 (Opening) = 침식→팽창

작은 노이즈 제거 + 원래 크기 유지. 비유: 돌기 깎기

닫힘 (Closing) = 팽창→침식

작은 구멍 메우기 + 원래 크기 유지. 비유: 구멍 메우기

히스토그램 평활화 + 감마 보정
cv.equalizeHist, 감마 보정 코드

히스토그램이란 이미지에서 각 밝기값(0~255)이 몇 개나 있는지 보여주는 그래프예요. 평활화는 이 분포를 균등하게 만들어서 대비를 높여요.

# 히스토그램 평활화 — 한 줄! result = cv.equalizeHist(gray) # 흑백 이미지만 가능! h = cv.calcHist([img], [2], None, [256], [0,256]) # ↑채널번호(R=2) ↑마스크없음 ↑256단계 ↑0~255범위

히스토그램 평활화 표 채우기 (31p — 시험 문제!)

감마 보정

각 픽셀에 지수(γ)를 적용해서 전체 밝기를 조절해요. γ < 1이면 밝아지고, γ > 1이면 어두워져요.

def gamma(f, gamma=1.0): f1 = f / 255.0 return np.uint8(255 * (f1 ** gamma)) # 반드시 np.uint8 변환! gc = np.hstack((gamma(img, 0.5), gamma(img, 1.0), gamma(img, 2.0)))

uint8 범위 초과 처리 — 39p ★별표

이미지는 기본으로 uint8 타입이에요. 0~255 범위만 저장 가능한데, 이 범위를 벗어나면 이상한 값이 생겨요!

에지(Edge)란?
밝기가 급격히 바뀌는 경계 부분

이미지에서 밝기가 급격하게 바뀌는 경계 부분이에요. 에지 강도는 √(Gx² + Gy²)로 계산해요.

밝기가 급격히 바뀌는 곳 = 에지. 에지 강도 = √(Gx²+Gy²)

Gradient 방향

밝기가 밝아지는 방향. arctan(Gy/Gx)로 계산. 시험에 안 나옴!

계산 효율

실시간 응용이 많아서 계산 속도가 중요해요.

모라벡 알고리즘
계산 문제 출제! S(0,1)=4 계산 과정

이미지에서 코너(모서리) 점을 찾는 알고리즘이에요. 상·하·좌·우로 조금 이동했을 때 밝기 변화가 모두 큰 점이 코너예요.

계산 방법

각 방향으로 1칸 이동했을 때 밝기 차이의 제곱합 S를 구해요.

  • S(오른쪽) = Σ (f(x,y) − f(x+1,y))²
  • S(왼쪽) = Σ (f(x,y) − f(x-1,y))²
  • S(아래) = Σ (f(x,y) − f(x,y+1))²
  • S(위) = Σ (f(x,y) − f(x,y-1))²

특징맵 값 = min(S오, S왼, S아래, S위)

최솟값을 쓰는 이유: 가장 약한 방향도 커야 진짜 코너!

특징점 = 특징맵에서 min(S) 값이 가장 큰 점!

특징점 판단: min(S)가 가장 큰 점들 = 어느 방향으로 가도 변화가 큼 = 코너!
해리스 특징점
표 5-1 암기! a=코너, b=에지, c=평탄

모라벡의 단점을 개선한 알고리즘이에요. 2차 모멘트 행렬을 이용해서 더 정확하게 코너를 검출해요.

점2차 모멘트 행렬C값판단의미
a[0.52,-0.2; -0.2,0.53]C=0.1925코너좋은 특징점!
b[0.08,-0.08; -0.08,0.8]C=0.0237에지특징점 아님
c[0, 0; 0, 0]C=0.0평탄특징점 아님

C 큰 양수=코너(특징점!) / C 음수=에지 / C≈0=평탄

기억법: 모든 방향으로 변화 큼=코너=C 큰 양수 / 한 방향만=에지=C 음수 / 변화 없음=평탄=C≈0
harris = cv.cornerHarris(gray, blockSize=4, ksize=3, k=0.04) # blockSize: 이웃 크기 / ksize: Sobel 커널 크기 / k: 보통 0.04 harris = cv.dilate(harris, None) # NMS용 팽창
