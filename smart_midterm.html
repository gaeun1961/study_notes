I2C 통신 원리 ★
SCL + SDA 2선 · 동기 · 브로드캐스트 방식

I2C: 마스터 1개 + 슬레이브 여러개 → SDA 한 선을 공유

I2C 통신 과정 (4단계) ★

① Start 신호 — 마스터가 통신 시작 선언
② 슬레이브 주소 7비트 전송 — 통신 대상 장치 선택 (모든 슬레이브가 듣지만 해당 주소만 응답)
③ R/W 비트
   0 → 마스터 → 슬레이브 (쓰기)
   1 → 슬레이브 → 마스터 (읽기)
④ ACK — 슬레이브가 수신 확인 응답
⚠️ SDA 선 하나를 공유하므로 한 번에 하나의 슬레이브만 통신 가능 → 상대적으로 느림
용도: 센서, 메모리, LED 디스플레이 등 저속 주변 장치
SPI 통신 원리 ★
4선 · 동기 · 전이중 · 고속

MOSI와 MISO 두 선 → 동시에 송수신 가능 (전이중)

I2C와 차이점
  • 선이 4개 → 송수신 동시 가능
  • I2C보다 빠름
  • 슬레이브마다 CS 핀 필요
SPI 통신 시작 조건
  • CS(CE) 핀 LOW → 통신 시작
  • CLK에 맞춰 1비트씩 송수신
  • CS HIGH → 통신 종료
UART 통신
Universal Asynchronous Receiver/Transmitter · 비동기
UART의 특징
  • 비동기(Asynchronous) 통신 — 클럭 없이 미리 정한 속도(baud rate)로 통신
  • 요청-응답 보장 없음 (버퍼에 쌓임 → 프린터 스풀처럼)
  • TX(송신)와 RX(수신) 2선 사용
⚠️ UART는 타이밍 보장이 안 됨 → 실시간성 필요한 센서엔 부적합
GPIO(General Purpose Input/Output)란? ★
범용 입출력 핀 — 라즈베리파이와 외부 장치를 연결하는 관문
정의

컴퓨터에서 범용 입력/출력 핀을 의미하며, 라즈베리파이 같은 단일 보드 컴퓨터에서 다양한 전자 부품과의 물리적 연결을 가능하게 하는 핀

  • GPIO 핀으로 센서, 모터, LED 등 외부 장치를 연결
  • 프로그래밍을 통해 데이터를 읽거나 장치를 제어
  • 디지털 신호 입력(읽기) / 출력(쓰기) 용도

GPIO.setmode()로 선택 · 헷갈리면 BCM 기준 외우기

폴링(Polling) 방식 ★
메인 루프에서 직접 핀 상태를 반복해서 확인
폴링이란?

메인 루프가 핀 상태를 계속 읽어가며 변화를 감지하는 방식. 버튼을 직접 감시함.

장점
  • 구현이 단순
  • 타이밍 직접 제어 가능
단점
  • CPU 낭비 (입력 없어도 루프 실행)
  • 메인 스레드 블로킹 (다른 작업 병행 불가)
  • 짧은 신호 누락 가능
  • 응답 지연 (polling 간격만큼)
이벤트(Event) 방식 ★
GPIO.add_event_detect() — 비동기 콜백
이벤트 방식이란?

특정 핀의 신호 변화를 비동기적으로 감지하여 자동으로 콜백 함수를 호출. 메인 루프를 점유하지 않음.

bouncetime=300 → 콜백 실행 후 300ms 동안 추가 이벤트 무시
PWM(Pulse Width Modulation)이란? ★
펄스폭 변조 — 디지털로 아날로그 효과 내기
정의

디지털 핀에서 ON/OFF 비율을 조절하여 아날로그처럼 동작하게 하는 방식

DutyCycle이 높을수록 평균 전력 ↑ → LED 밝기↑ / 버저 음색 변화

주파수(Hz)

1초당 PWM 신호가 반복되는 횟수
→ 소리의 높낮이 결정

DutyCycle(%)

한 주기에서 ON 상태 비율
→ LED 밝기 / 버저 음색 결정

PWM 응용주파수 역할DutyCycle 역할
🔊 버저음정(높낮이) 결정음색 (얇음↔두꺼움)
💡 LED깜빡임 속도밝기 결정
⚙️ 모터진동 주파수속도 결정
DutyCycle 상세 ★
PWM 한 주기에서 ON(HIGH) 상태가 차지하는 비율(%)
DutyCycle(%) = ON 시간 / 전체 주기 × 100
평균 전류 Iavg = I × DutyCycle   →   Iavg ∝ LED 밝기

버저(Buzzer) 종류와 원리
액티브 버저 vs 패시브 버저
액티브 버저
  • 내장 발진 회로 보유
  • 전원만 공급하면 특정 주파수로 소리

거리 측정 순서 ★
5단계 순서 — 시험 단골 문제!
① Trig에 10µs 이상 HIGH 신호 입력

→ 초음파 발사 시작 트리거

② 초음파 발사

→ 40kHz 초음파 8사이클(약 200µs) 생성 후 송신부(T)로 방사

③ Echo 핀 HIGH 전환 → 타이머 시작

→ 초음파 발사와 동시에 Echo 핀이 HIGH로 변환

④ 반사파 수신 → Echo 핀 LOW 전환 → 타이머 종료

→ 초음파가 물체에 반사되어 수신부(R)에 도달하는 순간

⑤ 거리 계산

→ Echo HIGH 유지 시간 = 초음파의 왕복 시간으로 거리 계산

Echo HIGH 구간 전체 = 초음파가 T(송신기)에서 출발해 물체를 거쳐 R(수신기)에 도달하는 왕복 시간

⭐ 순서 외우기: Trig HIGH → 초음파 발사 → Echo HIGH(타이머 시작) → 반사 수신 → Echo LOW(타이머 종료) → 거리 계산
거리 계산 공식 ★
음속 34300cm/s · 왕복 거리이므로 ÷2
거리(cm) = (Echo HIGH 시간(sec) × 34300) / 2
또는: 거리(cm) = Echo HIGH 시간(µs) / 58
왜 2로 나누나?

초음파가 센서 → 물체 → 센서(왕복)로 이동하므로 측정된 시간은 편도 거리의 2배. 실제 거리는 절반!

왕복 시간 t → 실제 거리 d = (t × 음속) / 2

  • 초음파 센서(HC-SR04)로 장애물 거리 측정
  • Trig → 발사 → Echo 시간 측정 → 거리 계산
  • 거리에 따라 경보음 제어
웨어러블/헬스케어
  • 심박 센서, 혈압 센서
  • 실시간 데이터 → 스마트폰 연동
  • 이상 감지 시 알림
⭐ 자동 주차 시스템: 초음파 센서 → Trig에 10µs HIGH → 초음파 발사 → Echo 시간 측정 → 거리 = (시간×34300)/2
LDR(광저항) 원리 ★
Light Dependent Resistor — 빛이 밝을수록 저항 감소

LDR은 빛을 받으면 저항이 줄어 전류가 잘 흐름. 소비 전압(AO)으로 밝기를 역으로 추정

  • 조도가 임계값 미만 → HIGH (어둠)

아날로그(AO) 출력
빛의 강도에 비례하는 연속 전압 신호
AO 특징
  • 빛의 강도에 따라 연속적으로 변하는 전압 출력
  • 세밀한 조도 측정 가능 (0~3.3V 사이 연속값)
  • 라즈베리파이는 아날로그 입력 불가 → 별도 ADC 필요
왜 ADC가 필요한가? ★
라즈베리파이의 한계와 MCP3208의 역할

아날로그 신호 → MCP3208(ADC) → SPI → 라즈베리파이

⚠️ 라즈베리파이는 GPIO가 디지털만 지원 → 아날로그 센서 직접 연결 불가 → MCP3208 같은 ADC 필요
SPI(Serial Peripheral Interface)란? ★
4선 직렬 동기 통신 · 전이중 · 고속
SPI 정의
  • 마스터가 생성한 CLK(클럭) 신호에 맞춰 비트 단위로 동기 통신
  • MOSI / MISO 두 선 → 동시에 송수신 가능 (전이중 Full-Duplex)
  • I2C보다 빠름 (선이 분리되어 있으므로)
  • 슬레이브마다 CS(Chip Select) 핀 필요
라즈베리파이에서 SPI 사용하려면: sudo raspi-config → Interface → SPI → Enable → Reboot

MCP3208이란? ★
12비트 8채널 ADC — 아날로그를 디지털로
기본 스펙
  • 12비트 해상도 → 0~4095 (2¹²-1)
  • 8채널 (CH0~CH7) → 8개 아날로그 장치 연결 가능
  • 인터페이스: SPI
  • 동작 전압: 2.7V ~ 5.5V
왜 필요?
  • 라즈베리파이는 아날로그 입력 없음
  • 아날로그 센서(조도·온도 등) → MCP3208 → SPI → 라즈베리파이
  • 1개로 최대 8개 아날로그 장치 연결
⚠️ 빵판에 꽂을 때 반드시 중앙 홈을 넘어서! 안 넘으면 양쪽 다리 연결 → 합선 → 칩 파손 (만원짜리!)
24비트(3바이트) SPI 프로토콜 ★
마스터가 3바이트 보내면 슬레이브도 3바이트 동시 반환
⭐ 핵심: 마스터가 3바이트 보내는 동안 슬레이브도 동시에 3바이트 반환. 응답의 마지막 12비트가 실제 ADC 값!
CMD 만드는 법 (CH0 기준) ★
채널 번호 → D2·D1·D0 비트 분해 → 3바이트 커맨드 조립
채널 → D2·D1·D0 분해
  • CH0 = 000 → D2=0, D1=0, D0=0
  • CH5 = 101 → D2=1, D1=0, D0=1
  • CH7 = 111 → D2=1, D1=1, D0=1

CMD 조립 (CH0 예시)

CMD비트 구성설명
cmd10b00000110 | D2더미5 + Start(1) + SGL(1) + D2(0) = 0x06
cmd2(D1 << 7) | (D0 << 6)D1·D0를 7·6번 비트에 배치 → CH0=0x00
cmd30x00전부 더미 (슬레이브 응답 받기 위한 클럭 유지)
Start bit = 1

통신 시작 신호

SGL/DIFF = 1

1 = 단일(Single) 모드 / 0 = 차동(Differential) 모드

응답 데이터 파싱 ★
adc[0]·adc[1]·adc[2] 에서 12비트 추출
응답 바이트내용처리
adc[0] (1st)전부 더미/Null무시
adc[1] (2nd)상위 4비트 Null + B11~B8adc[1] & 0b00001111 → 앞 4비트 마스킹
adc[2] (3rd)B7~B0 (하위 8비트)그대로 사용
12비트 조합 공식
high = adc[1] & 0b00001111 # 상위 4비트 추출 (앞 4개 Null 제거) high = high << 8 # 하위 8자리 비워두기 data = high | adc[2] # 하위 8비트 결합 → 최종 12비트 값
결과: data = 0~4095 사이의 정수 (밝을수록 낮음, 어두울수록 4095에 가까움)
전압 변환 공식
ADC 값(0~4095) → 실제 전압(V)
전압(V) = ADC값 × VREF / 4095
예시 계산
  • ADC값 = 3583, VREF = 3.3V
  • 전압 = 3583 × 3.3 / 4095 ≈ 2.89V
  • → 약간 어두운 환경 (3.3V에 가까울수록 더 어두움)
단일(Single) vs 차동(Differential) ★
노이즈가 많은 환경에서는 차동 모드로 노이즈 제거
단일(SGL=1)
  • CH0 하나만 사용
  • AO 신호 직접 측정
  • 짧은 배선 환경에서 충분
  • 실습에서 사용하는 방식
차동(DIFF=0)
  • CH0(신호+노이즈) - CH1(GND+노이즈)
  • = 순수 신호 (노이즈 상쇄)
  • 긴 배선(3~5m↑)에서 필요
  • 자동주차 센서 등 실외 설치 시

긴 배선이 없는 실습 환경에서는 단일 모드로 충분

⭐ 차동 모드: CH0(신호+노이즈) - CH1(노이즈) = 순수 신호. 수 미터 이상 긴 배선에서 사용
