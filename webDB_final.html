왜 정보공유가 필요한가? ★ 실제 출제
중간고사 실제 출제 문항 — 서술 답안 외워두기
HTTP는 비연결형(Stateless) 통신이다. 요청-응답이 끝나면 연결이 끊기고, 이전 페이지의 정보가 사라진다.

왜 문제가 되는가?

로그인 페이지에서 아이디를 입력하고 로그인 버튼을 눌렀다.

→ LoginServlet이 실행되어 아이디 정보를 처리했다.

→ 그런데 다음 페이지(마이페이지)로 이동하면 아이디 정보가 사라진다!

→ HTTP가 비연결형이기 때문에 이전 request 객체가 소멸되기 때문.

모범 답안: "HTTP가 비연결형(Stateless) 통신이기 때문에 페이지 이동 시 이전 요청 정보가 소멸된다. 따라서 로그인 상태 유지, 장바구니 등 여러 페이지에 걸쳐 데이터를 공유하려면 별도의 정보 공유 방법이 필요하다."

① URL Rewriting ★
URL 뒤에 ?key=value 형태로 데이터 전달

메소드

response.sendRedirect(String url)
인자: url — 이동할 URL (쿼리스트링 포함 가능) | 반환: void
request.getParameter(String name)
인자: name — 쿼리스트링의 키 이름 | 반환: String (없으면 null)
코드
// 보내는 쪽 String name = request.getParameter("name"); response.sendRedirect("ResultServlet?name=" + name + "&gender=남자"); // 받는 쪽 String name = request.getParameter("name"); String gender = request.getParameter("gender");
장점

구현 간단

단점

URL에 노출됨. 한글은 URLEncoder 필요.

② Cookie ★
서버가 클라이언트(브라우저)에 저장. name=value 형식.

메소드 — 생성/전송

new Cookie(String name, String value)
인자: name — 쿠키 키 / value — 값 | 반환: Cookie 객체
cookie.setMaxAge(int seconds)
인자: seconds — 유효시간(초). 0이면 즉시 삭제. 음수면 브라우저 닫을 때 삭제 | 반환: void
response.addCookie(Cookie cookie)
인자: cookie — 전송할 쿠키 객체 | 반환: void

메소드 — 읽기/수정

request.getCookies()
인자: 없음 | 반환: Cookie[] — 모든 쿠키 배열. 쿠키 없으면 null
cookie.getName() / cookie.getValue()
getName() → String (키 이름) | getValue() → String (값)
cookie.setValue(String value)
인자: value — 변경할 값 | 반환: void
전체 코드
// 생성 & 전송 Cookie ck = new Cookie("user", "kim"); ck.setMaxAge(3600); // 1시간 response.addCookie(ck); // 읽기 Cookie[] cookies = request.getCookies(); if (cookies != null) { for (Cookie c : cookies) { if (c.getName().equals("user")) { String val = c.getValue(); } } } // 삭제: setMaxAge(0) Cookie ck = new Cookie("user", ""); ck.setMaxAge(0); response.addCookie(ck);
장점
  • 서버 메모리 안 씀
  • 브라우저 닫아도 유지 가능
단점
  • 클라이언트 저장 → 보안 취약
  • 4KB 용량 제한
③ Session ★
서버에 저장. 브라우저마다 세션 ID 발급.

메소드

request.getSession()
인자: 없음 (또는 boolean — false면 없을 때 null 반환) | 반환: HttpSession
session.setAttribute(String name, Object value)
인자: name — 키 / value — 값 (Object 타입!) | 반환: void
session.getAttribute(String name)
인자: name — 키 이름 | 반환: Object — 형변환 (String) 필수! 없으면 null
session.removeAttribute(String name)
인자: name — 삭제할 키 | 반환: void — 단일 속성 삭제
session.invalidate()
인자: 없음 | 반환: void — 세션 전체 삭제 (로그아웃)
session.setMaxInactiveInterval(int seconds)
인자: seconds — 유효시간(초). 기본 1800초(30분) | 반환: void
전체 코드
HttpSession session = request.getSession(); session.setAttribute("loginUser", "kim"); // 저장 String user = (String) session.getAttribute("loginUser"); // 읽기 (형변환!) session.removeAttribute("loginUser"); // 단일 삭제 session.invalidate(); // 전체 삭제 (로그아웃) session.setMaxInactiveInterval(1800); // 유효시간 30분
세션은 브라우저 단위! 크롬에서 로그인해도 엣지로 열면 로그아웃. 방문 횟수처럼 지속돼야 하는 건 쿠키!
④ ServletContext ★

메소드

ctx.getAttribute(String name)
인자: name — 키 | 반환: Object — 형변환 필요
전체 코드
ServletContext ctx = request.getServletContext(); ctx.setAttribute("notice", "서버 점검 예정"); String notice = (String) ctx.getAttribute("notice");
웹 애플리케이션 단위로 하나. 모든 사용자, 모든 서블릿이 같은 공간 공유. Java의 static 변수와 유사.

JSP란? ★
Ch.6 Section 01
⭐ JSP = Java Server Pages = HTML 안에 자바 코드를 직접 사용할 수 있는 구조

서블릿의 단점(HTML을 out.print()로 문자열로 써야 함)을 보완하기 위해 만들어짐.

서블릿의 단점
out.print("<html><h2>" + name + "</h2></html>"); // HTML 태그를 문자열로 → 복잡!
JSP의 해결
<html> <h2><%= name %></h2> </html> // HTML 안에 자바 코드 삽입!

JSP의 특징

  • HTML 파일에 자바 코드를 직접 사용
  • 최초 호출 시 서블릿으로 변환되어 컴파일됨
  • 모든 JSP 페이지는 자바 클래스 형태로 관리
  • 컨테이너에 의해 메모리에 적재됨
  • CSS, JavaScript 사용 형식은 HTML과 동일
JSP 실행과정 ★
Ch.6 Section 01

최초 요청 (서블릿 클래스 없을 때)

① 클라이언트가 hello.jsp 요청 ② WAS: 서블릿 클래스 유무 확인 → 없음 ③ hello.jsp → hello_jsp.java (자바 소스로 변환) ④ hello_jsp.java → hello_jsp.class (컴파일) ⑤ jspInit() 호출 → 서블릿 실행 준비 ⑥ _jspService() 호출 → 요청 처리 ⑦ 처리 결과를 클라이언트에 응답

두 번째 이후 요청 (서블릿 클래스 있을 때)

① 클라이언트가 hello.jsp 요청 ② WAS: 서블릿 클래스 있음 + 메모리에 있음 ③ 바로 _jspService() 호출 → 요청 처리
서블릿 생명주기와 동일! jspInit()=init() / _jspService()=service() / jspDestroy()=destroy()
MVC 패턴
Ch.6 Section 01

소프트웨어 개발 디자인 패턴. 역할을 분리해 유지보수를 쉽게 함.

Controller — 서블릿

클라이언트 요청 받아 처리 흐름 제어. Model에 처리 요청 후 View로 전달.

View — JSP

화면 표시 담당. 데이터를 HTML로 렌더링.

Model — Business Logic

실제 데이터 처리(DB 조회 등). 비즈니스 로직 담당.

JSP 장단점
Ch.6 Section 01
장점
  • HTML에 자바 코드 직접 사용 → 서블릿보다 편함
  • 서블릿으로 변환되어 실행 → 서블릿의 장점 모두 가짐
  • MVC 패턴, 스프링 프레임워크 적용 가능
  • 커스텀 태그 라이브러리 사용 가능
단점
  • UI 변경해도 JSP→Java→class→실행 과정 거쳐야 해서 확인에 시간 소요
  • 화면 디자인 확인 시 반드시 서블릿 컨테이너 실행 필요

page 지시어 ★
Ch.6 Section 02 — JSP 파일 맨 앞에 위치. 이클립스에서 자동 생성.
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8" import="java.util.*" errorPage="error.jsp" %>
속성역할
language스크립트 언어 지정 (보통 "java")
contentType응답 파일 형식 + 인코딩 지정
pageEncoding컨테이너가 처리할 때 쓰는 인코딩. POST면 request.setCharacterEncoding()도 추가 필요
import자바 클래스 패키지 import. 여러 개면 콤마로 구분
errorPage에러 발생 시 이동할 페이지 지정
import 사용 예
<%@ page contentType="text/html" pageEncoding="utf-8" %> <%@ page import="java.util.Date" %> <% request.setCharacterEncoding("utf-8"); String uid = request.getParameter("id"); String res = "<h3>아이디: " + uid + "</h3>"; res += "<br>로그인 시간: " + (new Date()).toString(); out.print(res); %>
include 지시어 ★
Ch.6 Section 02 — 다른 파일을 현재 위치에 포함
<%@ include file="파일경로" %>
  • 사용된 위치에 특정 파일(html, jsp)의 내용을 그대로 가져옴
  • 컨테이너는 포함된 파일을 하나로 합쳐서 자바 소스 생성 후 컴파일
  • 원하는 위치에 자유롭게 사용 가능
헤더/바디/푸터 분리 예
<!-- main.jsp --> <%@ include file="header.jsp" %> <%@ include file="body.jsp" %> <%@ include file="footer.jsp" %> <!-- 컨테이너는 세 파일을 하나로 합쳐서 처리 -->
include 지시어로 포함된 파일들은 하나의 서블릿으로 변환됨 → 변수 공유 가능!
taglib 지시어
Ch.6 Section 02

JSP의 태그 확장인 커스텀 태그를 사용하기 위한 지시어.

<%@ taglib (uri="태그 라이브러리 경로" 또는 tagdir="태그 파일 경로") prefix="접두어" %>
속성역할
uri태그 라이브러리 위치 (.tld 파일 경로)
tagdir태그 파일로 구현한 경우 태그 파일 경로
prefix해당 태그를 구분하기 위한 접두어
사용 예
<%@ taglib tagdir="/WEB-INF/tags" prefix="m" %> <h2><m:printData /></h2>
스크립트릿 <% %> ★
Ch.6 Section 03 — 가장 기본. 모든 자바 코드 사용 가능.

서블릿 변환 시 _jspService() 메소드 내부에 배치됨. HTML과 섞어서 사용 가능.

기본 사용
<html> <% String uid = request.getParameter("id"); String res = "<h3>User ID: " + uid + "</h3>"; out.print(res); %> </html>
출력: out.print() 또는 out.println(). println()은 소스에서만 줄바꿈. 브라우저 줄바꿈은 <br> 태그 필요!
선언문 <%! %> ★
Ch.6 Section 03 — 전역 변수/메소드 선언

서블릿 변환 시 _jspService() 메소드 외부에 배치됨 → 전역으로 동작.

선언문 태그 <%! %>
  • 변수 + 메소드 선언 가능
  • _jspService() 외부에 배치
스크립트릿 태그 <% %>
  • 변수만 선언 가능 (메소드 불가)
  • _jspService() 내부에 배치
전역 변수 선언
<%! int count = 0; %> <body> <% out.println(++count); %> </body>
전역 메소드 선언
<%@ page import="java.util.Date" %> <%! String getDate() { return (new Date()).toString(); } %> <% out.print("로그인 시간: " + getDate()); %>
표현식 <%= %> ★
Ch.6 Section 03 — 값을 바로 HTML에 출력

out.println()의 인자로 적합한 모든 자바 코드 가능. 세미콜론(;) 없음!

<!-- 변수/메소드 출력 --> <h3>User ID: <%= request.getParameter("id") %></h3> 현재 시간: <%= java.time.LocalDateTime.now() %> <!-- 연산 결과 출력 --> <% int a=10; int b=20; int c=30; %> 합계: <%= a + b + c %> <!-- 결과: 60 -->
세미콜론(;) 쓰면 오류! <%= result %> O / <%= result; %> X
